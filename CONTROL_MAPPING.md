# Control Mapping

This repository uses public, synthetic control mappings for review only.

| Framework | Example Control Area | Wrapper Evidence |
|---|---|---|
| SOC 2 | change control / incident response | decision receipt, refusal reason, review posture |
| ISO 27001 | access control / operational control | authority basis, evidence basis, audit packet |
| NIST CSF | protect / detect / respond | consequence posture and replay status |
| NIST AI RMF | govern / map / measure / manage | AI-origin flag, boundary decision, escalation path |
| HIPAA | administrative safeguards | clinical corridor only; synthetic demonstration |
| GDPR | lawful basis / data movement | privacy corridor only; synthetic demonstration |
| DORA | ICT risk / operational resilience | financial or cyber corridor only |
| NIS2 | critical infrastructure risk | cyber corridor only |

## Rule

These mappings are not certifications.

They are evidence alignment examples that help reviewers understand what the boundary decision proves.
