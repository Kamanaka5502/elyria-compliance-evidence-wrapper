# Regulated Review Path

1. Confirm the source receipt is synthetic or authorized for review.
2. Confirm the runtime boundary decision is preserved.
3. Confirm the wrapper did not alter the decision.
4. Confirm the source receipt reference is present.
5. Review the compliance posture.
6. Verify the packet.
7. Determine whether controlled diligence is required.

## Controlled diligence trigger

Escalate to controlled review if the reviewer needs production receipt details, private replay review, customer-specific control mapping, deployment architecture, or protected runtime implementation.
